## Dark / Light Mode

Gerenciado pelo **next-themes**. Regras importantes:

- `<html className="dark" suppressHydrationWarning>` no layout garante dark mode desde o primeiro render, sem flash de tela branca.
- O `ThemeProvider` tem `defaultTheme="dark"` e `attribute="class"` — adiciona/remove a classe `dark` no `<html>`.
- **Não usar** `body { background-color: ... }` no globals.css — interfere com as classes Tailwind. O fundo do body é controlado só pelo Tailwind: `bg-zinc-50 dark:bg-zinc-950`.
- Todo componente usa o prefixo `dark:` para as variantes escuras.
- **Toggle na navbar:** abordagem CSS sem estado `mounted` — `<Sun className="hidden dark:flex" />` + `<Moon className="flex dark:hidden" />` — evita hydration mismatch.

## Internacionalização PT / EN

Implementada com **Context API puro** — sem next-intl ou i18n externo.

### Como funciona

1. **`lib/i18n.ts`** — objeto `translations` com `as const` contendo todas as strings de UI em `pt` e `en`. Exporta também o tipo `Lang = "pt" | "en"`.
2. **`lib/language-context.tsx`** — `LanguageProvider` com `useState<Lang>("pt")`, lê o idioma salvo no `localStorage` via `useEffect` após o mount, e persiste ao trocar. Exporta `useLanguage()`.
3. **Todos os componentes** importam `useLanguage()` e `translations[lang]` para exibir as strings corretas.
4. **Navbar** tem dropdown desktop "PT ▾ / EN ▾" e botões PT/EN no menu mobile.

### Trocar idioma no código

```tsx
const { lang, setLang } = useLanguage(); // hook nos Client Components
const t = translations[lang].hero;       // acessa a seção correta
```

### Typewriter e troca de idioma

O hook `useTypewriter` recebe `text` como dependência do `useEffect`. Quando o idioma muda, `t.hero.typewriter` muda → o efeito reinicia automaticamente com `setDisplayed("") + setDone(false)`.

## Como o formulário de contato funciona

```
Usuário preenche o form (contact-form.tsx — Client Component)
        ↓
useTransition chama sendContactEmail (app/actions.ts — Server Action)
        ↓
Zod valida: name ≥2, email válido, subject ≥3, message ≥10 chars
        ↓
Resend envia para CONTACT_EMAIL com reply_to do remetente
        ↓
Retorna { success: true } ou { success: false, error: string }
        ↓
UI: spinner Enviando... → verde (sucesso) ou vermelho (erro)
```

> `RESEND_API_KEY` fica **apenas no servidor** — nunca exposta no client bundle.

## Regras que devem ser seguidas sempre

1. **Backend só em `app/actions.ts`** — nunca lógica de servidor em Client Components.
2. **Tipagem estrita:** sem `any`, sem `@ts-ignore`. Rodar `npm run build` para confirmar.
3. **Nova seção:** criar em `components/sections/nome.tsx`, importar em `app/page.tsx`, adicionar strings em `lib/i18n.ts` nos dois idiomas.
4. **Ícones:** exclusivamente `lucide-react` — sem SVG inline.
5. **Animações:** `framer-motion` com `whileInView` e `viewport={{ once: true }}`.
6. **Dark mode:** sempre usar `dark:` prefix nas classes. Nunca hardcodar cor sem a contraparte dark (e.g. `bg-white dark:bg-zinc-900`). Confirmar que `className="dark"` está no `<html>`.
7. **Fontes:** `font-mono` (JetBrains) para títulos/badges/labels, `font-sans` (Inter) para texto corrido.
8. **i18n:** toda string visível ao usuário deve vir de `translations[lang]`. Nunca hardcodar texto diretamente nos componentes (exceto o nome "Lais Rodrigues" no hero/navbar).
9. **Vídeos:** `autoPlay muted loop playsInline preload="none"` + `useRef + useEffect + .play().catch(()=>{})`. Sem `hidden md:block` — vídeos aparecem em todos os dispositivos.
10. **Responsividade:** mobile-first, testar mentalmente em 375px / 768px / 1280px. Grid do formulário: `grid-cols-1 sm:grid-cols-2` com `min-w-0` nos wrappers e `overflow-hidden` no container.
11. **Copy:** direto ao ponto — máximo 2 frases por parágrafo, títulos curtos, sem floreio.
